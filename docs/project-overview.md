# Project Overview — oracledb (dart-oracledb)

> Generated by the Document Project workflow (deep scan), 2026-06-12. Primary entry

## Purpose

`oracledb` is a **pure-Dart Oracle Database driver** that implements Oracle's
thin-mode **TNS/TTC wire protocol** directly over TCP sockets. It requires **no
Oracle Client / Instant Client installation** and has **no native or FFI
dependencies** — everything (network framing, message encoding, cryptographic
authentication, data-type codecs) is implemented in Dart.

It mirrors the structure of the official **node-oracledb thin driver** to keep
the port maintainable and to allow upstream behaviour to be cross-checked
file-by-file (see `reference/` and the architecture doc).

- **Package name:** `oracledb` (repo `dart-oracledb`)
- **Version:** `0.9.2` (pre-1.0; see [Project Status](#project-status))
- **License:** see `LICENSE`
- **Repository:** https://github.com/abibiano/dart-oracledb

## What it does today

| Capability | Status |
|------------|--------|
| Connect over TCP and TLS/SSL | ✅ |
| Authentication — FAST_AUTH (Oracle 23ai) and classical AUTH_PHASE_ONE/TWO (pre-23) | ✅ |
| SELECT queries with column metadata, multi-round FETCH | ✅ |
| Bind parameters (named `:name` and positional `:1`) | ✅ |
| DML — INSERT / UPDATE / DELETE with row counts | ✅ |
| Transactions — `commit` / `rollback` / managed `runTransaction` | ✅ |
| PL/SQL — stored procedures, functions, OUT / IN OUT binds | ✅ |
| Statement caching (cursor reuse, parse skipping) | ✅ |
| Data types — NUMBER, VARCHAR2/CHAR, DATE, TIMESTAMP, TIMESTAMP WITH TIME ZONE, RAW, CLOB, BLOB, native JSON (OSON) | ✅ |
| Connection health check (`ping`) | ✅ |
| Connection pooling | ⏳ Planned (Epic 5, backlog) |

**Networking:** `dart:io` TCP sockets (`SecureSocket` for TLS).

## Platform Support

Supported native platforms: **Linux, macOS, Windows, Android, iOS**.

**Web is deliberately excluded:** `dart:io` is unavailable in the browser, and
`dart2js` maps Dart `int` to a JS double — silently losing precision past 2^53,
which would corrupt Oracle `NUMBER` / `rowid` values.

## Supported Oracle Versions

The auth path is **server-driven, not version-driven** — it is selected from the
`TNS_ACCEPT_FLAG_FAST_AUTH` bit in the server's TNS ACCEPT packet, never by
parsing version strings.

| Target | Image | Service | Auth path | CI / dev port |
|--------|-------|---------|-----------|---------------|
| **Oracle 23ai** (primary) | `container-registry.oracle.com/database/free` | `FREEPDB1` | FAST_AUTH (single round trip) | `1521` |
| **Oracle pre-23 (12c+)** (supported fallback) | `gvenzl/oracle-xe:21` | `XEPDB1` | classical AUTH_PHASE_ONE/TWO | `1522` |

> **Project rule:** every feature and bug fix must pass integration tests on
> **both** 23ai and 21c before it is considered complete. See
> [development-guide.md](./development-guide.md).

## Architecture at a glance

A clean **three-layer** design (transport → protocol → public API). See
[architecture.md](./architecture.md) for the full breakdown.

```
        Public API           OracleConnection, OracleResult/OracleRow,
   (lib/oracledb.dart)        OracleBind/OracleOutBinds, OracleTimestampTz,
            │                 OracleException, TlsConfig
            ▼
        Protocol layer        TTC messages (execute, auth, fast_auth, lob_op,
     (lib/src/protocol/)       ping, protocol), codecs (buffer, data_types,
            │                  oson), bind_parser, sql_classifier, statement_cache
            ▼
        Transport layer        TNS handshake (CONNECT/ACCEPT/REFUSE/REDIRECT),
     (lib/src/transport/)       TTC byte-stream send/receive loop, sequence
            │                   counter, poison-on-timeout, BREAK/RESET MARKER,
            ▼                   piggyback queues, sockets + TLS
        dart:io TCP / TLS  ───────────────────────────────────────────────►  Oracle DB
```

## Repository structure (high level)

- `lib/oracledb.dart` — single public export surface.
- `lib/src/` — all implementation (private): `transport/`, `protocol/`, `crypto/`, plus `connection.dart` and the public value types.
- `test/src/` — unit tests mirroring `lib/src/` (no DB required).
- `test/integration/` — real-DB integration tests (env-driven, run on both Oracle versions).
- `example/example.dart` — runnable usage example.
- `docker-compose.yml` / `docker/` — local Oracle 23ai + 21c containers.
- `reference/` — node-oracledb reference sources used during porting.
- `_bmad-output/` — BMad planning & implementation artifacts (PRD, architecture, epics, story specs, retros).

See [source-tree-analysis.md](./source-tree-analysis.md) for the annotated tree.

## Project Status

Pre-1.0 (`0.9.2`). Core connectivity, query/DML/transaction, PL/SQL, and the full
scalar + LOB + JSON data-type surface are implemented and validated against both
Oracle 23ai and 21c. The main remaining pre-1.0 epic is **Connection Pooling**
(Epic 5). Detailed status lives in
`_bmad-output/implementation-artifacts/sprint-status.yaml` and the per-epic
retrospectives.

**Known limitations** (see README for the authoritative list): single in-flight
`execute()` per connection (overlapping calls are rejected, not serialized — use
separate connections); a 1,000-FETCH-round-trip safety cap per execute surfaces
as `OracleResult.moreRowsAvailable`; thin-mode feature scope (no `DRCP`,
advanced queuing, etc.).
