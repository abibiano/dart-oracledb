## Project Overview

- **Type:** Monolith — single-part **Dart library** (pure-Dart Oracle Database driver)
- **Package:** `oracledb` · **Version:** `0.9.2` (pre-1.0)
- **Primary language:** Dart `^3.12.0` (pure Dart, no FFI, no web)
- **Architecture:** Three-layer protocol stack — public API → `protocol/` (TTC messages + codecs + OSON) → `transport/` (TNS + sockets/TLS) → `dart:io`
- **What it is:** Implements Oracle's thin-mode TNS/TTC wire protocol directly; **no Oracle Client required**. Mirrors the node-oracledb thin driver structure.

## Quick Reference

- **Tech stack:** Dart 3.12+, `pointycastle` (crypto), `logging`, `meta`; dev: `test`, `lints`
- **Public entry point:** `lib/oracledb.dart` → `OracleConnection`
- **Targets:** Oracle **23ai** (FAST_AUTH, port 1521/`FREEPDB1`) and **pre-23 / 21c** (classical auth, port 1522/`XEPDB1`) — auth path chosen from the server's ACCEPT flag, not version strings
- **Golden rule:** every change must pass integration tests on **both** Oracle versions

## Getting Started

1. `dart pub get` then `dart analyze` (must be zero warnings).
2. Unit tests need no database: `dart test test/src/`.
3. For integration tests, bring up **both** Oracle containers and run the suite
   against 1521 (23ai) and 1522 (21c) — see [Development Guide](./development-guide.md).
4. To use the driver, start from `lib/oracledb.dart` and the
   [API Reference](./api-reference.md); `example/example.dart` is a runnable demo.
