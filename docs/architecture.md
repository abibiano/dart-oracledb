## 1. Architectural style

A **layered (vertically sliced) protocol stack** with strict directional
dependencies:

```
public API  ──►  protocol  ──►  transport  ──►  dart:io socket
```

Each layer only depends downward. The public API never touches sockets; the
transport never decodes business values. Cross-cutting concerns — endianness,
Oracle error propagation, resource cleanup, crypto, and Dart↔Oracle type
conversion — are handled consistently at the layer that owns them.

The package structure deliberately **mirrors node-oracledb's thin driver**
(`sqlnet/` → `transport/`, `protocol/` → `protocol/`, `protocol/messages/` →
`protocol/messages/`) so behaviour can be cross-checked against the reference.

## 2. Layer responsibilities

### 2.1 Transport layer — `lib/src/transport/`

Owns the **TNS network protocol** and the **TTC byte stream**.

| File | LOC | Responsibility |
|------|-----|----------------|
| `transport.dart` | ~2715 | The `Transport` class: TNS handshake, TTC send/receive loop, sequence counter, poison mechanism, piggyback queues, EXECUTE/FETCH/COMMIT/ROLLBACK/LOB RPC orchestration |
| `socket.dart` | ~374 | `OracleSocket` — `dart:io` Socket wrapper: connect-with-timeout, async buffered reads, TLS upgrade, graceful `close()` vs forcible `destroy()` |
| `tls.dart` | ~63 | `TlsConfig` value type: `enabled`, `verifyCertificate`, optional `SecurityContext` |
| `packet.dart` | ~498 | TNS packet framing — 8-byte header, packet types, big-endian length, large-SDU (4-byte length, v315+), CONNECT builder, ACCEPT parser |
| `connect_string.dart` | ~130 | EZ-Connect parser (`host[:port]/service`) → `ConnectionInfo` |

**Key transport mechanisms (the non-obvious ones):**

- **Handshake:** `sendConnectReceiveAccept()` drives CONNECT → ACCEPT/REFUSE/REDIRECT/RESEND with a retry loop. `supportsFastAuth` is parsed from the ACCEPT packet's flag2 field (`0x10000000`) and **defaults `true`** so a parse miss leaves the 23ai path intact.
- **TTC stream loop:** `sendData()` frames the TTC payload into one or more TNS DATA packets (fragmenting at the negotiated SDU), applying the END_OF_RPC data flag (`0x0800`) on 23.4+ servers; bootstrap RPCs (protocol negotiation, auth) override to `0x0000`. `_receiveAllTtcData()` reads DATA packets until a **completion probe** (`ttcStreamIsComplete` for executes, the LOB variant for LOB ops) reports end-of-response.
- **Sequence counter:** a one-byte TTC message sequence that wraps `255 → 0 → 1`. Pinned inside `Transport` (a wrap-at-256 integration smoke test samples it across 300+ executes).
- **Poison-on-timeout:** Dart's `Future.timeout` does not cancel a pending socket read. On RPC timeout the transport calls `_poison()` to destroy the socket and flag itself unusable, so a late server response can never be misframed onto the next RPC. Subsequent calls fail fast with `ORA-03113` (distinct from the timeout error).
- **BREAK/RESET MARKER:** Oracle sends a MARKER (type 12, BREAK) for events such as constraint violations; the client must reply with a RESET MARKER before continuing to read DATA. Marker packets count against the receive-loop cap to prevent a flood bypass.
- **Piggyback queues:** pending cursor-close and free-temp-LOB messages are prepended to the next EXECUTE (no standalone RPC), chunked to stay within an SDU half-budget.

### 2.2 Protocol layer — `lib/src/protocol/`

Owns **TTC message encoding/decoding** and **Oracle data-type codecs**.

**Codecs:**

| File | Responsibility |
|------|----------------|
| `buffer.dart` | `ReadBuffer`/`WriteBuffer` over `Uint8List`. **Explicit-endianness only** (`readUint16BE`, `writeUint32LE`, …). Oracle variable-length integers (`ub1/ub2/ub4/ub8`, `sb1..sb8`: size byte + big-endian payload). Length-prefixed & 0xFE-chunked byte fields. A dedicated `BufferUnderflowException` means "need more packets" (drives completion probes); any other `BufferException` is fatal malformation |
| `data_types.dart` | Oracle wire-type codecs: **NUMBER** (base-100, variable length, 1–22 bytes), **DATE** (7-byte), **TIMESTAMP** (11-byte = DATE + 4-byte nanos BE), **TIMESTAMP WITH TIME ZONE** (13-byte UTC instant + zone bytes), **VARCHAR2/CHAR** (UTF-8; `'' == NULL`), **RAW**. Plus `inferOraTypeForValue` for IN-bind type inference |
| `oson.dart` | OSON (Oracle binary JSON, wire type 119) encoder/decoder. Emits OSON v1; decodes v1/v3 field names. **Fails loud** on Oracle-specific scalar nodes (DATE, INTERVAL, VECTOR, …) rather than silently nulling |
| `constants.dart` | All protocol constants: `oraType*` type codes, TTC function/message ids, execute-option flags, bind directions, LOB op codes, capability flags, field-version thresholds (12.2 / 20.1 / 23.1 / 23.4) |
| `bind_parser.dart` | Extracts `:name` / `:1` placeholders from SQL (respects string literals) |
| `lob_locator.dart` | `LobLocator` — decoded 40-byte locator + prefetch metadata; UTF-16BE helpers for var-length-charset CLOBs |
| `capabilities.dart` | Compile/runtime capability negotiation bits |
| `ttc_packet.dart`, `protocol.dart` | TTC packet wrapper and a small TTC state machine |

**Messages — `lib/src/protocol/messages/`:** all extend `Message` (`base.dart`: `messageType`, `sequence`, `encode(WriteBuffer)`, `toBytes()`).

| Message | Role |
|---------|------|
| `protocol_message.dart` | Version / driver-name negotiation |
| `auth_message.dart` | Classical AUTH_PHASE_ONE (`0x76`) and AUTH_PHASE_TWO (`0x73`) request/response; `VerifierParams` extraction; shared TTC-error parser |
| `fast_auth_message.dart` | 23ai single-message envelope bundling Protocol + DataTypes + AUTH_PHASE_ONE |
| `execute_message.dart` (~1767 LOC) | The central RPC: `ExecuteRequest` (parse/bind/execute/fetch options, `al8i4` array, DEFINE mode for LOB prefetch, long-data ordering, OSON binds), `ColumnMetadata`, `BindDir`/`BindMetadata`, `ExecuteResponse`, and the `ttcStreamIsComplete` / `decodeExecuteResponse` decode entry points; `FetchRequest` |
| `lob_op_message.dart` | LOB READ / WRITE / CREATE_TEMP / FREE_TEMP / GET_LENGTH |
| `ping_message.dart` | Single-function health check |

## 3. Cross-cutting concerns

- **Endianness:** TNS/TTC is mixed-endian. The buffer layer exposes only explicitly-named BE/LE methods; ambiguous `readUint16()` does not exist. (Headers are big-endian; many TTC fields are little-endian.)
- **Error propagation:** server failures arrive as TTC ERROR messages; they are decoded into `OracleException` carrying the `ORA-NNNNN` code, message, and (for query errors) the SQL text + character `offset`. `cause` always preserves the originating error.
- **Resource cleanup:** every resource has an explicit `close()`; `withConnection`/`runTransaction` wrap it in try/finally. A poisoned transport is destroyed, not reused.
- **Security:** passwords are never logged or stored — only derived keys. Credential failures are surfaced as a sanitized "invalid username or password" while real Oracle diagnostics are preserved internally.
- **Type conversion:** a single Dart↔Oracle mapping in `data_types.dart` / `oson.dart` is used for both binds and column decode, keeping encode/decode symmetric.

## 4. Authentication design

`AuthFlow.authenticate()` (`lib/src/crypto/auth.dart`) branches on
`transport.supportsFastAuth`:

- **FAST_AUTH path (23ai):** `transport.sendFastAuth()` sends Protocol + DataTypes + AUTH_PHASE_ONE in one round trip; AUTH_PHASE_TWO follows with `dataFlags: 0x0800`.
- **Classical path (pre-23):** `sendProtocolNegotiation()`, then AUTH_PHASE_ONE (`dataFlags: 0x0000`), then AUTH_PHASE_TWO (`dataFlags: 0x0000`).

Two logical phases always complete (`AuthState`: notStarted → phaseOneSent →
phaseTwoSent → authenticated/failed; state is never left mid-flight on failure).

**Crypto (all `pointycastle`, never `package:crypto`):**

- **Verifier dispatch** (`verifier.dart`): `0x939` = O5LOGON / SHA-512 (11g+); `0xB92`/`0x4815` = O8LOGON / PBKDF2-SHA512 (12c+).
- **Password key:** PBKDF2-HMAC-SHA512 over the salt for the 12c path (then `SHA512(key + AUTH_VFR_DATA)` sliced to 32 bytes = AES-256 key); a direct salted SHA-512 for the 11g path.
- **Session key:** decrypt server `AUTH_SESSKEY` (AES-256-CBC, zero IV) → server partial; generate a `Random.secure()` client partial; mix both via PBKDF2-SHA512 (`AUTH_PBKDF2_CSK_SALT`) → combo key.
- **Speedy key:** 12c path only.
- **Password proof:** random 16-byte salt prefix + password, AES-256-CBC under the combo key, hex-encoded.
- A short AUTH_PHASE_TWO timeout (default 5s) is interpreted as a silent password rejection, so wrong passwords fail fast instead of hanging.

## 5. Connection execution model

- **One in-flight `execute()` per connection.** A single TTC byte stream multiplexes one socket; overlapping executes would interleave writes and cross-read responses, corrupting cursor ids, the piggyback queue, and the sequence counter. The driver **rejects** a concurrent second call rather than serializing it. Concurrency requires separate connections (a future pool).
- **Statement caching:** cursor ids are reused and re-parses skipped for repeated SQL (instrumented via `debugFullParseExecutes` / `debugReuseExecutes`). PL/SQL blocks are deliberately not cached.
- **FETCH safety cap:** at most 1,000 FETCH round trips per execute; if the server still has rows, `OracleResult.moreRowsAvailable` is set and `rows` is a truncated prefix.

## 6. Key design decisions & rationale

| Decision | Rationale |
|----------|-----------|
| Pure Dart, no FFI | Cross-platform (incl. Android/iOS), no Oracle Client install, AOT-friendly |
| `pointycastle`, not `package:crypto` | Needs AES-256-CBC + PBKDF2 + SHA-512 primitives `package:crypto` doesn't provide |
| Structure mirrors node-oracledb thin driver | Byte-level cross-checking against a reference; easier upstream sync |
| Auth path from ACCEPT flag, not version string | Robust to version-string variance; matches how the server actually advertises capability |
| Web excluded | `dart2js` loses NUMBER precision past 2^53 |
| Reject overlapping executes | Cheaper and safer than serializing a single-stream socket |
| Fail-loud decoders | A malformed/unsupported wire value raises immediately rather than corrupting results silently |

## 7. Testing architecture

- **Unit tests** (`test/src/`, mirrors `lib/src/`) — no database; cover codecs, buffers, message encode/decode, crypto, classifier, cache.
- **Integration tests** (`test/integration/`) — against a real Oracle in Docker, env-driven via `test_helper.dart` (`ORACLE_HOST/PORT/SERVICE/USER/PASSWORD`); the **same suite runs on 23ai and 21c**. Auth-path-specific tests auto-skip on the wrong server using a `supportsFastAuth` probe.
- **CI:** `integration` (23ai) and `integration-21c` jobs; coverage floor enforced.
