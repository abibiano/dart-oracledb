## Entry points

| Concern | Start here |
|---------|-----------|
| Using the driver | `lib/oracledb.dart` (exports) → `lib/src/connection.dart` |
| Connecting / handshake | `connection.dart` `connect()` → `transport/transport.dart` `connect()` + `sendConnectReceiveAccept()` |
| Authentication | `crypto/auth.dart` `AuthFlow.authenticate()` |
| Running a query/DML | `connection.dart` `execute()` → `protocol/messages/execute_message.dart` |
| Decoding a value | `protocol/data_types.dart` (scalars) / `protocol/oson.dart` (JSON) |
| Wire bytes in/out | `protocol/buffer.dart` + `transport/packet.dart` |

## Critical directories — purpose

- **`lib/src/transport/`** — everything below the TTC message boundary: sockets, TLS, TNS framing, the byte-stream loop, failure (poison) semantics. Touch with care; protocol-correctness bugs here are subtle and version-dependent.
- **`lib/src/protocol/`** — the codec heart. `buffer.dart` (endianness) and `data_types.dart` (NUMBER base-100, date/timestamp formats) are the most reused and the easiest to get wrong.
- **`lib/src/protocol/messages/`** — one class per TTC message; `execute_message.dart` is by far the largest and most central.
- **`lib/src/crypto/`** — `pointycastle`-only auth crypto; `package:crypto` must never be added.
- **`test/integration/`** — the real correctness gate. `test_helper.dart` is mandatory for connection params (no hardcoded `FREEPDB1`/`XEPDB1`/ports/creds).
- **`reference/`** — node-oracledb sources; the canonical answer when a wire-format question arises.
