## What Went Well

### Architecture Review Paid Immediately

The Epic 9 preparation task — architecture review of buffer.dart and fast-auth charset negotiation — was completed before Epic 10 story authoring. The thin server-conversion model decision (no client-side charset codecs, server converts, client writes UTF-8 always) was made once and held through all five stories without revision. This is the pattern we want for any epic that touches a protocol-level assumption.

### `forceUncacheable` Seam — Invisible Post-Connect Detection

Story 10.1's startup detection query needed to be invisible to the existing statement-cache instrumentation. The `forceUncacheable` seam through `_executeGuarded` and `_prepareStatement` solved this cleanly. The integration assertion — `debugCacheSize`, `debugFullParseExecutes`, `debugReuseExecutes`, and `debugPendingCloseCount` all zero after connect — proved it. This is the right level of rigor for any internal seam that touches connection lifecycle.

### Thin Model Held End-to-End

Story 10.2's integration test round-tripped ASCII, accented Latin, CJK, and emoji through a `VARCHAR2` column via the bind path on both AL32UTF8 fixtures. Story 10.3 did the same on a real WE8MSWIN1252 database. The server-conversion model is now proved, not just asserted.

### National Charset Reference Alignment (Story 10.4)

AC1 and AC5 in Story 10.4 were amended during code review because the original ACs described a csfrm model that didn't match the in-repo node-oracledb reference. The review caught it. The fix — align to the reference, not an assumed model — was the right call. The reference-first discipline continues to prevent architectural drift.

### Story 10.5 — 13 Dismissals

The adversarial review of Story 10.5 produced 13 dismissed findings and 4 real patches. A high dismissal count on a docs/CI story means the panel was working hard and finding nothing real. That's a healthy signal.

### Adversarial Review Caught Real Issues Every Story

- 10.1: Null PARAMETER column coalesced to empty string instead of throwing; stack trace loss on non-OracleException errors; credential-free test missing terms.
- 10.2: `capabilities.dart` `defaultCharset` missing doc cross-reference; "Capabilities are trimmed" test offset off-by-one (coincidental pass — `0x03 == 3`).
- 10.3: WE8MSWIN1252 guard too broad; non-ASCII test literals not Unicode-escaped; missing `RUN_INTEGRATION_TESTS` gate; session-detach wait missing.
- 10.4: Diagnostics missing actual charset; national binds not gated before wire on unsupported charset; odd-byte UTF-16BE not rejected; NULL national values failing silently; enum index compatibility; debug internals in CLOB mismatch errors.
- 10.5: Stale comment; missing `set -e`/`set -o pipefail` in CI wait loop; README CLOB row missing cross-ref; "local-only" label contradiction.

Twenty-two patches. None cosmetic.

## Challenges

### CI Coverage Floor — Stale but Not Dangerous

The floor is at 68%, last measured (68.89%) on 2026-06-10 during Epic 7. Current measured coverage is **73.81%** — three epics of test work moved it up organically. The floor isn't dangerously low; it's just stale on the low side. Recalibrating to ~73% or higher would make the gate meaningful again and provide a tighter signal before Epic 11 adds a large new surface.

### Story 10.3 — Most Patches, Sequencing Surprise

Story 10.3 had 6 patches — the most of any Epic 10 story. The session-detach wait before charset migration was the most notable: it was a runtime sequencing requirement that wasn't in the original design. This is characteristic of "first real non-AL32UTF8 execution" stories — the theory had been proven by 10.1 and 10.2, but 10.3 was the first story to actually exercise the non-AL32 wire path end-to-end against a real database. The sequencing issue would not have been visible from the architecture review.

### Deferred Items Propagate Forward (10.1/10.2 → 10.3/10.4)

Stories 10.1 and 10.2 accumulated 5 deferred items between them (caps length byte overflow guard, classical DataTypes response parser alignment, `ttcCharsetAl16Utf16 = 2000`, etc.). The `ttcCharsetAl16Utf16` deferral in particular meant Story 10.4 had to introduce both the constant *and* the behavior in one story, contributing to its 6-patch review load. When a constant is deferred from the story where it first makes conceptual sense, the receiving story carries more scope than it should.

### Story 10.4 AC Alignment Required Mid-Review Amendment

The csfrm model for national character types was not fully specified in the story ACs before dev began. The review caught that AC1/AC5 described a model inconsistent with the in-repo reference. This required amending the ACs during review rather than before development. The architecture review for Epic 10 covered buffer.dart and fast-auth negotiation; it didn't go deep enough on the NCHAR/NVARCHAR2 bind/define csfrm model. Pre-story architecture reviews need to cover the full protocol surface the story will touch, not just the headline concern.

## Key Insights

1. **Architecture reviews must cover the full protocol surface, not just the headline concern.** The buffer.dart + fast-auth review was correct but incomplete — the csfrm bind/define model for national types was not reviewed and had to be corrected mid-story-10.4 review. For Epic 11, the executeMany architecture review must cover array bind vectors, BATCH_EXECUTE message shape, and DML RETURNING array results.
2. **Deferring a constant to a later story can increase that story's scope and review load.** `ttcCharsetAl16Utf16 = 2000` deferred from 10.2 to 10.4 meant 10.4 introduced constant + behavior together. Consider whether deferring a constant "for documentation parity" is worth the cost.
3. **`ping()` TTC-stream desync was fixed the same day as the Epic 9 retrospective** (`702cc20`, 2026-06-21). The Epic 9 retro artifact predated the commit by hours; the fix was real and complete with a dual-env regression test. Retrospectives should check git history before carrying forward prior action items.
4. **The 68% CI coverage floor is stale but not dangerous.** Actual current coverage is 73.81% (measured from existing lcov.info). The floor was set at 68% based on a June 10 measurement; three epics of test work moved the actual number up. Recalibrate to ~73% before Epic 11 to keep the gate meaningful.
5. **The thin server-conversion model is the right architecture.** No client-side charset codecs, no NLS_LANG dependency, server does the work, client writes UTF-8. This held cleanly through five stories and a real WE8MSWIN1252 fixture. It should be referenced as a precedent for any future globalization work.
6. **Session-detach wait in Story 10.3 is a runtime sequencing pattern to document.** When a test migrates connection NLS settings (e.g., `ALTER SESSION SET NLS_CHARACTERSET`), the session must be detached and re-acquired before the new setting takes effect. This is not obvious and will appear again in any future NLS-mutation test.

## Next Epic Preview: Epic 11 — Bulk DML (`executeMany`)

**Epic 11 goal:** Array DML binding for high-volume insert/update/delete and PL/SQL.

**Dependencies on Epic 10 work:** None direct. Charset negotiation and national type handling are parallel concerns.

**Preparation concerns:**

- `executeMany` sends array bind vectors in BATCH_EXECUTE TTC messages. This is a different message shape from the current scalar-bind EXECUTE path. `execute_message.dart` will need either a new message class or a substantial branch.
- The node-oracledb reference implementation covers executeMany in `connection.rst:1629` and `appendix_a.rst:176`. Winston should walk through the wire format before Story 11.1 ACs are authored.
- `ping()` desync is resolved (`702cc20`). Epic 11 integration tests may use `conn.ping()` safely.
- DML RETURNING with executeMany (Story 11.4) returns arrays of OUT values — a new result shape not currently handled.

**Significant discovery assessment:** No findings from Epic 10 require changes to Epic 11's direction. Epic 11's plan is sound pending its protocol architecture review.

Winston (Architect): "The BATCH_EXECUTE message shape is not a minor extension. The current execute_message.dart was designed for scalar binds. Array bind vectors, per-row error semantics, and DML RETURNING arrays are all protocol-level additions. The architecture review is not optional."
