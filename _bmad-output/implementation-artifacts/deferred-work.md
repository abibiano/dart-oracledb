## Resolved: probe underflow/malformation distinction (2026-06-12)

- `ttcStreamIsComplete` treated EVERY `BufferException` as "need more packets", including non-underflow malformations (`Integer too large`, sign-bit rejection in `_readInteger`) ‚Äî resolved: a new `BufferUnderflowException extends BufferException` is thrown only by `ReadBuffer._checkAvailable` (the one true out-of-bytes signal); both completion probes (`ttcStreamIsComplete`, `lobOpStreamIsComplete`) return false only on the underflow subtype and rethrow any other `BufferException` as `OracleException(oraProtocolError)` with the malformation as `cause`; the transport's `isComplete` closure poisons the connection before propagating a probe throw (framing-desync hazard ‚Äî same pattern as the AC10 cap and mid-query REFUSE).
